# go-Redis 高并发抢票系统（Go + Gin 后端开发）
**技术栈**：Go 1.21+、Gin、GORM、MySQL、Redis (Redis Cluster/Lua)、Redis Stream、JWT、Docker

## 项目亮点
- 🌐 **高并发秒杀架构**：基于 Redis Lua 脚本实现原子性库存扣减，结合 Redis Stream 消息队列完成异步削峰与数据库落库，解决高并发下的超卖、数据不一致及数据库瓶颈问题。
- 🔒 **全链路安全防护**：接入 JWT 鉴权与分布式锁，实现用户唯一性校验与防重复购买，保障业务数据安全。
- ⚡ **高性能压测表现**：在 500 并发用户、每秒 200 启动速率的压测场景下，实现 **RPS 峰值 932**，平均响应时间 **468ms**，库存与订单数据一致性 100%。

## 核心职责
1. **架构设计与开发**：主导秒杀系统后端架构设计，实现从用户下单、库存扣减到异步落库的全流程逻辑。
2. **核心功能开发**：基于 Gin + GORM 开发用户、活动、订单、门票四大核心模块，完成 CRUD 与业务逻辑封装。
3. **高并发解决方案**：使用 Redis Lua 脚本实现原子性库存预扣减，通过 Redis Stream 削峰填谷，异步写入数据库。

## 技术细节 & 成果
### 1. 秒杀核心实现
- **Redis Lua 原子操作**：将库存扣减逻辑封装于 Lua 脚本，确保在高并发下库存扣减的原子性，**杜绝超卖**。
- **Redis Stream 异步写入**：订单与门票数据不直接写入 MySQL，而是推送至 Redis Stream，由消费者异步处理，**大幅提升接口响应速度**。
- **分布式锁与防重**：结合用户 ID 与活动 ID 生成分布式锁，**拦截重复购买请求**，保障用户权益。
